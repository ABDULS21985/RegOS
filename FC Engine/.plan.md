21. Eliminate Every Inline Style — Extract to Semantic CSS Classes

Across the admin portal, there are ~20+ instances of style="display:flex; gap:...", style="background:#b91c1c; color:#fff;", style="font-weight:500;" and similar inline styles baked directly into .razor files (confirmed in TenantList.razor lines 142, 146, and throughout filter toolbars). This is an architectural defect — not a minor code smell. Do the following: (1) Audit every .razor file in Components/Pages/ and Components/Shared/ for any style= attribute that contains CSS properties, (2) For each, create the appropriate CSS class in app.css using the existing design token variables (e.g., background: var(--clr-danger) instead of #b91c1c), (3) Replace all inline styles with these class names, (4) Group new classes under their component section in app.css (e.g., /* ===== TENANT LIST ===== */), (5) Replace the two hardcoded action button inline styles in TenantList with class="fc-btn fc-btn--danger fc-btn--xs" and class="fc-btn fc-btn--success fc-btn--xs", (6) For filter toolbar layout, create .fc-toolbar, .fc-toolbar__filters, .fc-toolbar__actions classes with flexbox using --space-* tokens, (7) Verify zero style= attributes remain on any rendered HTML element (Blazor dynamic styles like style="--delay:@ms" for CSS custom property injection are acceptable).

22. Fix the Mobile Table Accessibility Defect (aria-hidden Cards)

In DataTable.razor, the mobile card view is rendered with aria-hidden="true" on the card container, which makes the entire table content invisible to screen readers on mobile — a WCAG 2.1 failure. The desktop <table> element is hidden via CSS on mobile but remains in the DOM, and the card view takes over. The fix requires: (1) Remove aria-hidden="true" from the mobile card container entirely, (2) Add aria-hidden="true" to the desktop <table> when in mobile view (detected via JS using ResizeObserver), (3) Give each card a role="row" and each data point within it role="cell" with aria-label="ColumnName: value", (4) Ensure the card's primary action (click to open) has role="button", tabindex="0", keyboard handlers for Enter/Space, and aria-label="Open [row title]", (5) The select checkbox in each card must have aria-label="Select [row title]", (6) The swipe-reveal action buttons must have proper aria-label attributes, not just icons, (7) Add aria-live="polite" to the .fc-dt-status-bar element so screen readers announce "12 results" when filters change, (8) Test with @media (max-width: 640px) and verify semantic correctness.
